Dining Note 자세히보기

target

다이닝 노트의 강점은 심플함 이지만 심플한 만큼의 정보의 부재가 단점입니다.
또 Dining Note는 음식의 기본 정보가 없으며 몸무게 입력기능이나, 통계 그래프가 없습니다.

단순히 먹는 음식만 입력하다 보니 어플의 본질성을 잃고, 단순 메모 어플로 변질될 가능성이 있습니다.

style guide

used skill

팀 프로젝트로 진행이 되어 함께 작업하기위해 figma로 기존 어플의 identity를 잃지 않기 위해 wireframe을 사용 하였습니다.

기존 어플에 없는 다양한 필요의 요소들을 넣기 위해 그래프 디자인을 사용하였습니다.

problem & solution

- Problem : 월 통계에 식사 횟수만 숫자로 표시되어 다이어트 목적에 필요성을 못느낌
- Problem : 기록 표시가 여러가지 항목으로 복잡한 화면 (기록에 대한 UI의 설명 부족)
- Problem : 사용성이 떨어지는 불필요한 요소들이 많습니다. (누구와 먹었는지, 어디서 먹었는지, 메모용 사진 업로드 기능)

-Solution : 하루 섭취 칼로리, 탄단지를 보기 쉬운 통계 그래프를 추가 하였습니다.
-Solution : 기록이 되어 있다는 표시를 월 캘린더에 표시되도록 디자인했습니다.
